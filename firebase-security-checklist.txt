FacturePro Firebase security checklist

1. Authentication
- Enable Email/Password provider.
- Add authorized domains:
  - facturergratuit.com
  - www.facturergratuit.com if used
  - localhost only for testing
- Enable email verification messaging.
- Confirm password reset email template uses the live domain.

2. Firestore
- Paste outputs/firestore.rules in Firestore Rules.
- Keep user data under users/{uid}/...
- Do not allow public read/write for account data.
- Store only the data needed for profile, history, projects, clients and products.

3. Cloud Storage
- Paste outputs/storage.rules in Storage Rules.
- Store account files under users/{uid}/...
- Allow only PDF and image files.
- Keep per-file upload size small, for example 10 MB.
- Use app/dashboard cleanup for old documents.

4. Account storage quota (800 KB)
- The whole account (profile/history/projects/clients/products) syncs to a
  single Firestore document at users/{uid}, which has a hard, non-negotiable
  1 MB (1,048,576 byte) limit per document - this is a Firestore platform
  limit, not something rules or app code can raise.
- Client UI enforces an 800 KB quota (ACCOUNT_STORAGE_QUOTA_BYTES in app.js)
  with a safety margin under that 1 MB ceiling, plus optional auto-cleanup.
- Per-image uploads (logo/stamp/signature) are capped client-side to stay
  well under this budget even across multiple saved companies.
- This is enforced entirely client-side; a technical user could bypass the
  JS check, but Firestore's own 1 MB document limit is the real backstop -
  any write beyond it is rejected by Firestore itself, not by our code.
- If genuinely large per-user storage (hundreds of MB) is needed later, the
  account data model needs to move from one big document to per-record
  Firestore subcollections (and large binary assets like logos into Firebase
  Storage instead of inline base64) - that is a real architecture change,
  not a config tweak.

5. App Check
- Enable App Check for Web app if available on your plan/settings.
- Use reCAPTCHA Enterprise only if the project plan supports it.

6. Privacy and compliance
- Keep privacy, cookies, terms, legal notice and contact links visible.
- Use a Google-certified CMP before serving personalized ads to EEA/UK/Switzerland users.
- Do not put server secrets, private API keys or OAuth tokens in index.html.
